# 代码转测试工作流

从**已有前后端代码**逆向生成**测试用例 + Python(Playwright) 自动化测试脚本**。专为"功能开发完了、测试严重缺失"的团队设计。

## 本技能性质

这是 **Workflow 型技能**（流程引导）。核心动作是**端到端跟做**——引导你走完从读代码到输出测试用例和脚本的完整流程，每个阶段有过门检查（Gate），过门才进下一步。

一份代码，同时产出两类交付物：

- **测试用例文档**（给人看）：结构化的测试用例表格，含编号、前置条件、步骤、预期结果、优先级
- **自动化测试脚本**（给机器跑）：基于 Playwright + pytest 的 Python E2E 脚本，可直接 `pytest` 运行

## 能解决什么问题

> 测试靠手点、没有回归测试、改了代码不敢发布。本技能解决"代码写完了，测试从零开始"的痛点。

核心思路：**代码即真相**。从代码中提取表单字段、校验规则、业务流程、状态流转，自动生成覆盖正常/异常/边界的测试用例和对应脚本。

## 浏览器能操作什么

> Playwright 可以操作浏览器中的一切用户可见元素。以下场景全覆盖：

| 操作类型 | 能不能测 | 说明 |
|---------|---------|------|
| 文本输入 | ✅ | input / textarea / 富文本 |
| 下拉选择（原生） | ✅ | `<select>` 用 `select_option` |
| 下拉选择（UI 组件库） | ✅ | Element UI / Element Plus 的自定义下拉，点击触发 + 点选项 |
| 日期/时间选择 | ✅ | 日期范围、时间选择器，需处理弹出的日历面板 |
| 文件上传 | ✅ | 单文件、多文件 |
| 表格操作 | ✅ | 分页、排序、行内编辑、行内按钮 |
| 弹窗/对话框 | ✅ | 确认弹窗、消息提示、抽屉、Modal |
| 复选框/单选/开关 | ✅ | checkbox / radio / switch |
| 表单校验提示 | ✅ | 断言校验错误消息是否出现 |
| 登录态保持 | ✅ | storage state 复用 |
| 多标签页/iframe | ✅ | 弹出窗口、嵌入页面 |

## 流程总览

```
阶段1 收集输入 → Gate1 → 阶段2 读代码+提取测试点 → Gate2 → 阶段3 输出用例+脚本 → Gate3
```

每个 Gate 都有过门检查清单，详见 `01-workflow.md`。
