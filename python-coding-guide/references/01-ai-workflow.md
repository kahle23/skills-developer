# AI 编码工作流

核心原则：**先找，再读，最后写。** 写代码前先搜项目已有实现。

---

# 一、搜索已有代码

| 需求类型 | 搜索方式 |
|---------|---------|
| 新增功能 | 搜索功能相关的函数名、类名、关键词 |
| 工具函数 | 搜索 `*util*.py`、`*helper*.py`、`*common*.py` |
| 数据处理 | 搜索 model、service 中是否有类似逻辑 |
| 枚举/常量 | 搜索 `*enum*.py`、`*const*.py`、`*constant*.py` |
| 接口定义 | 搜索路由定义、装饰器（`@app.route`、`@router`、`@bp`） |

搜索范围优先级：当前模块 → 相邻模块 → 公共模块（common/utils） → 第三方库。

---

# 二、阅读相关文档和代码

文档优先级：README → API文档 → 数据库设计文档 → 模块说明 → 本技能文档（references/）。

阅读已有代码时关注：命名风格、返回类型、异常处理方式、工具类选择、日志格式、类型提示风格，新代码须与已有风格保持一致。

---

# 三、判断复用策略

```
搜索结果分析
│
├─ 完全匹配 → 直接调用
├─ 相似但不完全匹配 →
│   ├─ 差异小 → 扩展已有函数（增加参数）
│   └─ 差异大 → 新建函数，但引用已有函数处理公共逻辑
├─ 可组合的多个函数 → 组合调用
└─ 完全没有 → 编写新代码，遵循已有风格
```

---

# 四、编写新代码

1. **遵循已有风格** — 参考已有代码的命名、异常处理、类型提示风格
2. **遵循技能规范** — 参考 `03-code-standards.md` 等文档
3. **使用项目已有工具模块** — 优先用项目已引入的库
4. **保持接口一致性** — 返回类型、参数风格与已有代码统一

---

# 五、记录与文档化

新增了通用函数或新模式时：代码注释说明用途；通用工具函数考虑放入公共模块；新的编码模式可通过 `02-code-examples-template.md` 沉淀到技能中。
