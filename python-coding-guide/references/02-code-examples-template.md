# 代码示例学习模板

用户提供代码示例时，按以下流程将其沉淀到技能文档中。

---

# 一、处理流程

```
收到用户代码示例
│
├─ Step 1: 分析 — 提取命名、结构、风格特征
├─ Step 2: 归类 — 判断属于哪个已有文档的范畴
├─ Step 3: 去重 — 检查是否与已有规则冲突或重复
├─ Step 4: 提炼 — 从具体代码中抽象出通用规范
└─ Step 5: 写入 — 按模板格式追加到对应文档
```

---

# 二、归类规则

| 示例特征 | 归入文档 |
|---------|---------|
| 命名、类结构、函数规范、文档字符串 | `03-code-standards.md` |
| 列表/字典/集合/元组操作、推导式 | `04-data-structures.md` |
| 接口抽象、装饰器、模式运用 | `05-design-patterns.md` |
| 性能相关写法 | `06-performance.md` |
| 并发、异步相关写法 | `07-concurrency.md` |
| 测试、虚拟环境、依赖管理 | `08-testing-and-tooling.md` |

---

# 三、写入模板

```markdown
### X.Y [规则名称]

\```python
# 精简示例代码，只保留能说明规则的部分
\```

[一句话说明规则要点]
```

---

# 四、去重与冲突

| 情况 | 处理方式 |
|------|---------|
| 与已有规则完全一致 | 跳过，不重复写入 |
| 与已有规则冲突 | 标注差异，询问用户以哪个为准 |
| 已有规则的补充 | 在已有规则下方追加补充段落 |
| 全新规则 | 按模板新增 |
