# Python Coding Guide — AI 编码指南技能

一个指导 AI 编写高质量、风格统一的 Python 代码的技能。面向 Python 3.9+，不绑定特定框架（Flask / FastAPI / Django 均适用）。

> 文档中部分示例使用了 3.9+ 的 `list[int]` 泛型语法、3.10+ 的 `X | Y` 联合类型。低版本项目用 `typing.List` / `typing.Union` 替代即可。

> **入口文件是 `SKILL.md`**，它包含完整的文档索引、触发钩子和按场景的加载规则。本 README 只介绍技能概述、定制方法和 FAQ。

<br />

## 这个技能能做什么？

安装此技能后，AI 在编写 Python 代码时会：

1. **先搜索已有代码** — 不会盲目新建函数，而是先在项目中查找是否已有类似功能可复用
2. **遵循统一的编码规范** — 命名、结构、文档字符串、异常处理、日志、类型提示风格保持一致
3. **善用 Pythonic 特性** — 推导式、dataclass、装饰器、上下文管理器、生成器
4. **应用设计模式** — 在合适场景使用策略、工厂、观察者、单例等模式
5. **注意性能** — 避免常见陷阱（list 查找、循环内重复计算、字符串拼接）
6. **正确处理并发** — I/O 密集用线程/异步、CPU 密集用进程
7. **编写测试** — 使用 pytest 编写规范的单元测试

## 如何定制这个技能

### 1. 调整编码风格偏好

如果你对某些规范有不同偏好，直接编辑对应文档即可。比如：

- **不喜欢某种命名风格** → 编辑 `03-code-standards.md`，修改命名示例
- **不想用 dataclass** → 删除或修改 `05-design-patterns.md` 中的相关章节
- **有特定的文档字符串格式** → 编辑 `03-code-standards.md` 中的文档字符串章节

<br />

### 2. 添加第三方库文档

如果项目使用了重要的第三方库（如 FastAPI、Pydantic、Celery 等），可新建文档记录用法：

```
步骤：
1. 在 references/ 目录下创建新文档，编号从 21 开始递增
   如：references/21-fastapi.md、references/22-pydantic.md
2. 按以下格式编写：安装 → 核心功能 → 常用API → 示例代码 → 注意事项
3. 更新 SKILL.md 中的文档索引表和加载规则表
```

<br />

### 3. 通过代码示例自动学习

你也可以直接给 AI 发送代码示例，它会自动提炼规范并写入技能：

```
步骤：
1. 把你认为风格好的代码片段发给 AI
2. AI 会按 02-code-examples-template.md 的流程分析代码
3. 提取出命名、结构、工具库使用等规范
4. 自动追加到对应的文档中
```

支持批量发送多个代码片段，AI 会提取共性规范。

<br />

### 4. 添加核心编码规范文档

如果需要补充新的编码规范（如 SQL 编写规范、API 设计规范等）：

```
步骤：
1. 在 references/ 目录下创建新文档，编号在 09~20 区间内递增（03~08 已占用）
   如：references/09-api-design.md、references/10-sql-standards.md
2. 按现有规范文档的格式编写（标题一分类 → 标题二小分类 → 代码示例）
3. 更新 SKILL.md 中的文档索引表和加载规则表
```

<br />

### 5. 删除不需要的文档

如果某些文档对你的项目不适用，直接删除即可：

```
- 不需要并发编程？删除 07-concurrency.md
- 不需要测试规范？删除 08-testing-and-tooling.md

删除后记得从 SKILL.md 的文档索引表和加载规则表中移除对应条目。
```

<br />

## 常见问题

**Q: 如何让 AI 优先使用项目中已有的工具方法？**

编辑 `01-ai-workflow.md` 中的搜索范围，将你的公共模块路径加入优先搜索列表。

**Q: 如何让 AI 不使用某些我不喜欢的写法？**

直接编辑对应文档，将不推荐的写法标注为 `# BAD`，并说明原因。

**Q: 技能文档太多，AI 会不会每次都读取所有文档？**

不会。AI 会根据 SKILL.md 中的加载规则表按场景查阅，不会每次都读取全部文档。
