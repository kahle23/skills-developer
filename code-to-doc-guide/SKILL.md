# 代码转文档指南

从**已有代码 + 截图 + 口头描述**逆向生成文档。专为"需求靠截图和口头沟通、文档严重滞后"的团队设计。

一份代码，同时产出两类文档：

- **技术文档**（给开发看）：技术设计、业务流程、接口、数据存储、关键逻辑
- **用户使用文档**（给最终业务用户看）：傻瓜式操作步骤、前置条件、常见问题

## 适用场景

> 团队是"草台班子"模式：需求 = 截图 + 口头沟通 + 简单文档 → 直接开发 → 文档缺失。
> 本技能解决"代码已经写完了，但文档从零开始"的痛点。

核心思路：**代码即真相**。当代码与口头描述冲突时，以代码实际行为为准。

## 工作流

1. 收集输入（代码路径/截图/口头描述）→ 2. 按代码阅读策略读代码 → 3. 反推业务逻辑与数据流 → 4. 识别缺口向用户确认 → 5. 选模板输出文档 → 6. 标注待补充项

详见 01。

## 加载规则

| 用户意图 | 读取 |
|---------|------|
| 梳理/生成任意文档 | 01 → 判断文档类型 → 02 或 03 |
| 只生成技术文档 | 01 → 02，再参考 99 技术文档部分 |
| 只生成用户使用文档 | 01 → 03，再参考 99 用户文档部分 |
| 不确定怎么读代码 / 代码量大不知从哪入手 | 04 |
| 看完整范例对照写法 | 99 |

## 关联技能

- `feature-design-workflow` — 功能**设计**文档（开发前写）。本技能是开发**后**从代码逆向，方向相反，互补使用。
- `java-coding-guide` — Java 编码规范，阅读后端 Java 代码时辅助理解框架约定（如 Baibao/Kunlun 的 Service 分层）。
- `db-design-guide` — 表结构设计规范，技术文档中"数据存储"章节涉及表结构时参考其命名/字段规范。
