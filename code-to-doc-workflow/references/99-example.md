用户说："帮我梳理资产领用功能的文档，代码在 `views/asset/borrow/` 和 `AssetController` 里。"

通过阅读代码，获得以下信息（略去阅读过程）：

- **后端**：`AssetController`（4 个接口）、`AssetService`（业务逻辑）
- **涉及表**：`oa_asset_record`（资产领用记录）

## 输出一：技术文档

### 资产领用 - 技术文档

#### 1. 功能概述

| 项 | 内容 |
|----|------|
| 功能名称 | 资产领用 |
| 功能简述 | 员工在线申请领用公司资产，管理员审批通过后完成领用，支持归还操作 |
| 涉及角色 | 普通员工（申请领用）、资产管理员（审批、库存管理） |
| 模块位置 | 前端：`views/asset/borrow.vue`；后端：`AssetController` / `AssetService` |

#### 2. 技术设计

**2.1 架构概览**

```
前端 Vue
  └── views/asset/borrow.vue        # 领用申请、记录列表
        └── api/asset.js            # 封装 4 个接口
              └── 后端
                    ├── AssetController    # 4 个接口入口
                    └── AssetService       # 业务逻辑 + 库存扣减
                          └── oa_asset_record（资产记录表）
```

**2.2 核心组件**

| 层级 | 组件 | 职责 |
|------|------|------|
| 前端页面 | `views/asset/borrow.vue` | 领用申请表单、领用记录列表 |
| 前端 API | `api/asset.js` | 封装 apply/approve/list/return 接口 |
| Controller | `AssetController` | 接收请求、参数校验 |
| Service | `AssetService` | 业务逻辑、库存扣减、状态流转 |
| 数据库表 | `oa_asset_record` | 领用记录持久化 |

#### 3. 业务流程

**3.1 主流程**

```
员工提交领用申请（选择资产 + 填写用途）
    → 校验资产是否可用（status=可用）
    → 创建领用记录（status=待审批）
    → 通知管理员审批
    → 管理员审批：
        ├─ 通过 → 扣减资产库存 → 记录状态=已领用 → 记录领用时间
        └─ 拒绝 → 记录状态=已拒绝 → 记录拒绝原因
    → 员工归还资产 → 恢复库存 → 记录状态=已归还
```

**3.2 状态流转**

```
待审批 (0)
  ├─ 审批通过 → 已领用 (1)
  │              └─ 归还 → 已归还 (2)
  └─ 审批拒绝 → 已拒绝 (3)
```

#### 4. 接口设计

| 接口 | 方法 | 路径 | 核心入参 | 说明 |
|------|------|------|---------|------|
| 领用申请 | POST | `/asset/borrow/apply` | assetId, purpose | 提交领用申请 |
| 领用审批 | POST | `/asset/borrow/approve` | recordId, isApproved, reason | 审批（通过/拒绝） |
| 领用记录 | GET | `/asset/borrow/list` | pageNum, pageSize, status(可选) | 分页查询领用记录 |
| 归还 | POST | `/asset/borrow/return` | recordId | 归还已领用的资产 |

#### 5. 数据存储

**表：`oa_asset_record`**

| 关键字段 | 类型 | 含义 |
|---------|------|------|
| id | bigint | 主键 |
| asset_id | bigint | 资产 ID |
| borrower_id | bigint | 领用人 |
| purpose | varchar(200) | 领用用途 |
| status | int | 状态：0=待审批, 1=已领用, 2=已归还, 3=已拒绝 |
| apply_time | datetime | 申请时间 |
| approve_time | datetime | 审批时间 |
| return_time | datetime | 归还时间 |
| approve_reason | varchar(500) | 审批意见/拒绝原因 |

> 其他标准字段（create_user, create_time 等）参考 db-design-standard 规范。

#### 6. 关键业务逻辑

**6.1 库存扣减逻辑**

```
审批通过时执行：
1. 查询资产当前可用库存（SELECT ... FOR UPDATE 行锁）
2. 判断可用库存 > 0
3. 扣减库存（可用量 -1）
4. 更新领用记录状态为"已领用"，记录领用时间
5. 事务提交

如果库存不足：回滚，提示"该资产已被领用"
```

**6.2 防重复领用**

```
提交申请前校验：
同一资产，如果存在状态为"待审批"或"已领用"的记录，
则不允许提交新申请，提示"该资产正在被领用中"
```

## 输出二：用户使用文档

### 资产领用 - 使用说明

#### 这个功能是干什么的？

资产领用功能让您在线申请领用公司资产（如电脑、办公设备等）。提交申请后由资产管理员审批，审批通过即可领取，用完可以在线归还。

#### 开始之前

使用本功能前，请确认：

- [ ] 您有系统账号，且有"资产领用"的权限
- [ ] 您要领用的资产在系统中已登记

> 如果看不到「资产领用」菜单，说明没有权限，请联系资产管理员。

#### 怎么领用资产

**第一步：进入资产领用**

1. 登录系统，在左侧菜单点击「**资产管理**」→「**资产领用**」

   > 📷 *[截图：系统主界面，红框标注左侧菜单的「资产管理」→「资产领用」]*

**第二步：提交领用申请**

1. 在资产领用页面，点击右上角「**申请领用**」按钮

   > 📷 *[截图：标注「申请领用」按钮位置]*

2. 在弹出的表单中填写：

   | 字段 | 怎么填 | 必填 |
   |------|--------|------|
   | 选择资产 | 从下拉列表选择要领用的资产 | ✅ |
   | 领用用途 | 填写为什么要领用（如"新员工入职配电脑"） | ✅ |

   > 📷 *[截图：领用申请表单]*

3. 确认信息后，点击「**提交**」

4. 看到"**申请提交成功，等待审批**"提示，说明已提交

   > 📷 *[截图：提交成功的提示]*

**第三步：查看审批结果**

1. 回到资产领用页面，在"我的申请"列表查看状态

   - 待审批：管理员还没处理，请等待
   - 已通过：可以去领资产了
   - 已拒绝：查看拒绝原因，可重新申请

   > 📷 *[截图：申请列表，标注状态列]*

#### 归还资产

资产用完后，需要归还：

1. 在"我的申请"列表，找到状态为"已领用"的记录
2. 点击「**归还**」按钮
3. 确认归还

   > 📷 *[截图：归还操作]*

#### 常见问题

**Q：提交申请后能取消吗？**
A：审批通过前，可以点击「撤销」取消申请。通过后不能取消，但可以归还。

**Q：提示"该资产正在被领用中"怎么办？**
A：说明这个资产有人正在申请或已领用，请选择其他可用资产。

**Q：审批被拒绝了怎么办？**
A：查看拒绝原因。如果是用途填写不清楚，修改后重新提交即可。
