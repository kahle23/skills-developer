## 写作要点

1. **以代码为真相**：代码怎么写的就怎么记，不要脑补代码没有的逻辑
2. **重点写"为什么"和"关键逻辑"**：流水账式的描述价值低，代码本身已经说明了调用关系
3. **流程图必须有**：业务流程用图比用文字有效 10 倍
4. **接口列核心的就行**：不用把所有 CRUD 都列出来，列业务核心接口
5. **主动标缺口**：读不出来的用 `[待补充]` 标注，不要硬编
