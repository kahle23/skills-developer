# AI 编码工作流

核心原则：**先找，再读，最后写。** 写代码前先搜项目已有实现。

---

# 一、搜索已有代码

| 需求类型 | 搜索方式 |
|---------|---------|
| 新增功能 | 搜索功能相关的类名、方法名、关键词 |
| 工具方法 | 搜索 `*Util.java`、`*Helper.java`、`*Common*.java` |
| 数据处理 | 搜索 Mapper、Service 中是否有类似查询 |
| 枚举/常量 | 搜索 `*Enum.java`、`*Const*.java`、`*Constant*.java` |
| 接口定义 | 搜索 API 文档、Swagger 注解、Controller 方法 |

搜索范围优先级：当前模块 → 相邻模块 → 公共模块（common/util） → 第三方工具库。

---

# 二、阅读相关文档和代码

文档优先级：README → API文档 → 数据库设计文档 → 模块说明 → 本技能文档（references/）。

阅读已有代码时关注：命名风格、返回类型、异常处理方式、工具类选择、注解风格、分页方式、日志格式，新代码须与已有风格保持一致。

---

# 三、判断复用策略

```
搜索结果分析
│
├─ 完全匹配 → 直接调用
├─ 相似但不完全匹配 →
│   ├─ 差异小 → 扩展已有方法（增加参数/重载）
│   └─ 差异大 → 新建方法，但引用已有方法处理公共逻辑
├─ 可组合的多个方法 → 组合调用
└─ 完全没有 → 编写新代码，遵循已有风格
```

---

# 四、编写新代码

1. **遵循已有风格** — 参考已有代码的命名、注解、异常处理风格
2. **遵循技能规范** — 参考 `03-code-standards.md` 等文档
3. **使用项目已有工具类** — 优先用项目已引入的工具库
4. **保持接口一致性** — 返回类型、参数风格与已有代码统一

---

# 五、记录与文档化

新增了通用方法或新模式时：代码注释说明用途；通用工具方法考虑放入公共模块；新的编码模式可通过 `02-code-examples-template.md` 沉淀到技能中。
