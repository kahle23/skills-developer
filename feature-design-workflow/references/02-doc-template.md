# 02 — 设计文档模板

文档必须包含以下五个章节。每章末尾有 **mini-gate**（章节级检查点），写完一章过一遍。

---

## 一、需求概述

- 功能背景与目标
- 适用范围说明（如：仅 IT 设备 或 全公司资产）
- 核心名词定义（如有业务专属术语）

**mini-gate**：

- [ ] 背景说清了"为什么做"，不只是一句话复述功能名
- [ ] 范围明确包含边界（适用对象 + 不适用对象）
- [ ] 有专属术语的，已在术语表中定义

---

## 二、功能模块梳理

先概述（共几个子模块、核心/扩展各几个），再按子模块逐个列小表格：

**模块概览**

一句话说明本功能包含哪些子模块、核心和扩展功能各几个。

**逐模块列出**

### {子模块名}（核心 / 扩展）

| 功能点 | 说明 |
|--------|------|
|        |      |

> 优先级通过小标题后的"核心/扩展"标记区分，便于一眼看清主次。

**mini-gate**：

- [ ] 有模块概览（总数 + 核心/扩展分布）
- [ ] 每个子模块都有"核心/扩展"标记
- [ ] 功能点说明具体（不是"管理设备"这种泛词，而是"录入设备并自动生成编码"）
- [ ] 没有遗漏明显功能点（如列表页缺了"查询筛选"）

---

## 三、表结构设计

### 3.1 新建表

### 3.2 复用现有表

| 表名 | 复用方式 | 说明 |
|------|----------|------|
|      | 直接读写 / 关联查询 |      |

### 3.3 数据字典预设（如需要）

提供项目字典表的初始化 SQL（示例表名 `sys_dict_group` / `sys_dict_item`，按项目实际字典表结构调整）。

**mini-gate**：

- [ ] 新建表的 SQL 完整可执行（不是伪代码）
- [ ] 表名前缀符合项目惯例
- [ ] 复用表已列出并说明复用方式
- [ ] 有枚举字段的，已在注释中标注取值含义（如 `status tinyint COMMENT '0=在库 1=已领用'`）

## 四、接口梳理

> 接口的请求方式、返回结构、异常码等遵循团队接口规范，本章节只梳理需要哪些接口及其核心字段。

按功能模块分组，每组一个表格：

| 序号 | 接口地址 | 接口说明 | 核心入参 |
|------|----------|----------|----------|
|      |          |          |          |

**mini-gate**：

- [ ] 第二章列出的每个功能点，都有对应接口（或明确标注"前端控制，无需接口"）
- [ ] 接口地址前缀与表名前缀保持一致
- [ ] 核心入参列全（不是"等字段"，而是明确列出必填入参）

---

## 五、关键业务逻辑说明

对复杂业务逻辑进行详细说明：

- **编码/编号自动生成规则**：格式、递增策略、重置条件
- **状态流转规则**：建议用文本流程图表示状态变更路径
- **异步处理逻辑**：导入/导出/生成任务的执行流程和异常处理
- **自动触发机制**：如用户禁用自动解除关联、状态联动更新
- **数据权限控制**：按机构/角色/数据所属人的可见范围
- **字段填写说明**：不同场景下哪些字段必填、哪些可选

> 不是每个功能都有全部 6 类逻辑——只写这个功能确实有的。但如果一个都没有，这章至少要有字段填写说明。

**mini-gate**：

- [ ] 编码有生成规则（如果功能涉及编码）
- [ ] 有状态字段的，状态流转规则已画出
- [ ] 有批量/异步操作的，执行流程已说明
- [ ] 字段填写说明覆盖了主要操作场景
