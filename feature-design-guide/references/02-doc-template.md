# 02 — 设计文档模板

文档必须包含以下五个章节。

## 一、需求概述

- 功能背景与目标
- 适用范围说明（如：仅IT设备 或 全公司资产）
- 核心名词定义（如有业务专属术语）

## 二、功能模块梳理

先概述（共几个子模块、核心/扩展各几个），再按子模块逐个列小表格：

**模块概览**

一句话说明本功能包含哪些子模块、核心和扩展功能各几个。

**逐模块列出**

### {子模块名}（核心 / 扩展）

| 功能点 | 说明 |
|--------|------|
|        |      |

> 优先级通过小标题后的"核心/扩展"标记区分，便于一眼看清主次。

## 三、表结构设计

### 3.1 新建表

### 3.2 复用现有表

|------|----------|------|

### 3.3 数据字典预设（如需要）

提供项目字典表的初始化 SQL（示例表名 `sys_dict_group` / `sys_dict_item`，按项目实际字典表结构调整）。

## 四、接口梳理

> 接口的请求方式、返回结构、异常码等遵循团队接口规范，本章节只梳理需要哪些接口及其核心字段。

## 五、关键业务逻辑说明

对复杂业务逻辑进行详细说明：

- **编码/编号自动生成规则**：格式、递增策略、重置条件
- **状态流转规则**：建议用文本流程图表示状态变更路径
- **异步处理逻辑**：导入/导出/生成任务的执行流程和异常处理
- **自动触发机制**：如用户禁用自动解除关联、状态联动更新
- **数据权限控制**：按机构/角色/数据所属人的可见范围
- **字段填写说明**：不同场景下哪些字段必填、哪些可选
