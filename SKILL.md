## 按需加载规则

根据任务类型决定读取哪些文档：

| 任务类型 | 需要读取的文档 |
|---------|---------------|
| 简单重构/抽取方法 | 无需读取，参考现有代码风格即可 |
| 新增服务/接口 | 03 |
| 新增 API 接口 | 03 → 08 |
| 使用特定工具库 | 对应工具库文档（21-25） |
| 性能优化 | 06 |
| 并发编程 | 07 |
| 复杂业务逻辑/设计模式 | 05 |
| 不确定如何编码 | 01 → 03 |

**任务复杂度判断**：

```
用户输入任务
│
├─ 是否涉及新领域/新技术？ ──是──→ 读取对应文档
│
├─ 项目已有类似代码可参考？ ──是──→ 直接参考现有代码，跳过文档
│
├─ 任务类型判断：
│   ├─ 简单操作（改名/抽取/移动/重命名） → 跳过
│   ├─ 中等任务（新增方法/接口/服务） → 读 03
│   └─ 复杂任务（架构设计/并发/性能） → 读对应文档
│
└─ 不确定 → 读 01（工作流）再判断
```

**简单任务特征**（跳过文档）：
- 抽取方法到新类
- 重命名变量/方法/类
- 移动代码位置
- 简单的 CRUD 操作
- 项目已有成熟模式可参考

**需要读取文档的场景**：
- 首次接触项目，不了解编码风格
- 使用不熟悉的工具库或框架
- 涉及并发、性能优化等专业领域
- 需要设计新的接口或服务架构
